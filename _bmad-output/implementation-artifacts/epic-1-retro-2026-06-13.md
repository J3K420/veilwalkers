## 2. What went genuinely well

- **The atomic-mutation pattern compounded across stories.** Story 1.4 established the pipeline: `validate → capture model ref once → mutate in-memory → persist → post-persist recovery-swap check → rollback on fault → release lock → raise events (after release)`. 1.5 cloned it, 1.7/1.8/1.9 reused it. By 1.9 the *deviations* from it (1.8's single two-field write; 1.9's deliberate **no-rollback** on the ad-cap counter to bound in-process grants) were conscious, documented design calls — not drift.
- **The shared lock decision landed at the right moment.** 1.4 used a per-service `SemaphoreSlim`. 1.5 — the story that introduced the *second* mutator — recognized the cross-service interleave hazard (one service's persist durably writing another's not-yet-committed delta) and promoted it to a single shared `SaveMutationLock`. The implicit rule held: *the story that introduces the second mutator owns the mitigation.*
- **Code review caught real correctness bugs, not just nits.** The two standouts:
  - **1.8 future-date guard:** `!= today` treated future-dated stamps as claimable, reopening a same-day double-claim window once the real day caught up. Fixed to `< today` (CompareOrdinal).
  - **1.9 unbounded in-process cap bypass (MAJOR):** the original AdHook rolled the in-memory `GrantsToday` counter back on a persist-fail yet returned `Granted=true`; against a persistently-failing disk, every call landed a real charge while the on-disk counter never advanced → unlimited charges. Fix: do **not** roll the counter back, so the next call reads the consumed cap from live `SaveService.Current`.
- **Seams reserved without over-building.** `IProgressStore` (→Firebase), `IClock`, `ITelemetrySink`/`ITelemetryStore`, `AdHook`, `IPurchaseVerifier` — all stubbed with honest contracts and tests, none gold-plated. 1.9 added zero schema changes and zero new asmdef edges.
- **The acyclic-graph guard got stronger as it was stressed.** 1.2 upgraded it from tier-rank ("no strictly-higher tier") to an explicit allowed-edge matrix (which also catches forbidden *sideways* edges); 1.4 added the sanctioned `Economy → Persistence` edge as a visible matrix row rather than a silent loosening.

## 3. Recurring patterns — the real lessons

### Lesson 1 — Doc-comments are contract surface, and they kept lying
The same class of finding recurred across 1.4, 1.5, and 1.6: a doc-comment overstated what the code actually did.
- 1.4: false "both events fire after persist" claim for `OnInsufficientCredits`; missing `OverflowException` in the throw enumeration; undocumented dirty-read window on the `Balance` getter. **Four of nine 1.4 patches were doc corrections.**
- 1.6: `OnValidate` doc claimed it surfaced "the same problems the `ProgressionRules` ctor enforces" — it only warned on two cases.

**Takeaway:** write the doc-contract true on the first pass. A false doc-comment is a defect a reviewer will (rightly) file, and it costs a patch + re-review cycle every time.

### Lesson 2 — Tautological tests passed for the wrong reason (twice)
- 1.4: `FakeProgressStore` returned the same reference it stored (`Loaded = model`), so "persisted value" assertions were vacuous. Fixed by cloning/snapshotting on save.
- 1.5: recovery-swap regression tests read through the swapped-in recovered model (`progression.Xp`), so they passed even with the rollback deleted. Fixed by capturing the pre-swap `SaveModel` reference and asserting directly on it. **Mutation-tested: deleting the rollback now turns them red.**

**Takeaway:** a test that cannot fail is worse than no test. The fake store must clone on save; every recovery/rollback test must assert on the captured pre-swap reference, and should be mutation-tested at least once.

### Lesson 3 — Unity headless testing has sharp, repeatable traps
- `LogAssert` fails any test that observes an `Error` log — this forced a *semantic* design decision (recoverable/handled states log `GameLog.Warn`, not `Error`; only unexpected failures log `Error`) and bit again via `ConfigureAwait(false)` continuations logging outside the main test context.
- Json.NET's default `DateParseHandling` silently converted the ISO-8601 `isoTimestampUtc` to a culture DateTime on round-trip — fixed with `DateParseHandling.None` on both the serializer settings *and* the `JsonTextReader` (the `JObject` parse site doesn't inherit settings).
- `Application.persistentDataPath` is main-thread-only — must be read in Bootstrap and passed in.
- UTF 1.1.33 has no `async Task` test support — tests block on `GetAwaiter().GetResult()`, which is deadlock-free only because `ConfigureAwait(false)` prevents sync-context capture.

(These are tracked in the `unity-headless-testing` memory — keep feeding it.)

### Lesson 4 — The architecture guard is itself load-bearing and evolving
The allowed-edge matrix in `AcyclicDependencyTests` is the canonical declaration of the dependency graph (reading `.asmdef` JSON, not `CompilationPipeline.GetAssemblies()`, which silently skips still-empty assemblies). New *legal* same-tier edges (e.g. `Economy → Persistence` in 1.4) require a deliberate matrix edit — which is the point: it makes every new edge visible and intentional.

### Lesson 5 — Two day-representations coexist by design
1.8 stores the claim date as a human-readable `yyyy-MM-dd` UTC string (`SaveModel.DailyClaim`); 1.9 stores ad-cap/first-zero buckets as a compact `long` (`DayBucket.For(utc) = utc.Date.Ticks / TicksPerDay`). Different shapes for different jobs, both UTC, both via the `IClock` seam — documented so a future reviewer doesn't try to "unify" them.
