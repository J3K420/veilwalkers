## Deferred from: code review of 3-1-disclose-and-request-camera-permission-gracefully (2026-06-15)

- **`Granted` camera-permission state is never re-validated, so a permission revoked while the app is backgrounded is missed** — `CameraPermissionView.OnApplicationFocus` only pumps `OnRequestResult()` while the flow is `Requesting` or `Denied` (the comment: "Granted/Disclosure need no re-query"). The 3.1 ACs (AC-1/2/3) do not include "permission stays valid after backgrounding," and decision #3 documents `Granted` as needing no re-query *within this story's boundary*. Owner: **Story 3.3** — before attempting AR entry / starting the session, re-validate camera permission (re-run `CameraPermissionFlow.Evaluate()` or an equivalent gate) and route back to the disclosure/re-grant flow if it was revoked; the `CameraPermissionFlow` already exposes `Evaluate()` for exactly this re-check.
