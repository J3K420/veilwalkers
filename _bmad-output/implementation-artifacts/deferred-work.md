## Deferred from: story-1.2 (2026-06-12)

- **EDM4U Android Gradle templates not committed (only the folder is)** — The Google External Dependency Manager (EDM4U), bundled with ARCore Extensions, auto-resolves Android dependencies and copies `mainTemplate.gradle` into `Assets/Plugins/Android/`. That folder is now committed as a tracked-but-empty stub (`.gitkeep` + `.meta`) so the resolver has a copy target — without it, every Android auto-resolve throws `DirectoryNotFoundException: …/Assets/Plugins/Android/mainTemplate.gradle` (first hit after Story 1.2). The EDM-**generated** templates themselves (`mainTemplate.gradle`, `gradleTemplate.properties`, `settingsTemplate.gradle`, `launcherTemplate.gradle`, `baseProjectTemplate.gradle` and their `.meta`s) are gitignored, so a fresh clone / CI / build machine relies on EDM regenerating them from the pinned packages (Assets → External Dependency Manager → Android Resolver → Force Resolve). Owner: AR-21 — when wiring the real Android build, decide whether to **commit** the resolved Gradle templates (reproducible, version-controlled build config — remove the matching `.gitignore` lines) or keep them generated-on-resolve; and confirm the resolved templates carry the AR/Play dependencies (ARCore, Play Billing) the build needs. Note: the active build target being Android is what triggers EDM auto-resolve, so this also recurs if a later story flips the build target — the committed folder prevents the crash either way. [.gitignore "Android Gradle templates generated by …" block; Assets/Plugins/Android/.gitkeep]
